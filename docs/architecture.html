Architecture Overview

How Spectra organizes rendering, windows, and IPC runtime modes.

High-Level Layers

API Layer

  • Easy API one-liners
  • Object API: App/Figure/Axes/Series
  • Python proxies over IPC

UI + Interaction

  • Input handling and gestures
  • Docking, tabs, split view
  • Command/undo/workspace systems

Rendering Backend

  • Vulkan command pipeline
  • Per-window swapchain contexts
  • Headless and on-screen rendering

Runtime Modes

ModeDescriptionWhen to Use
In-process UI, renderer, and plotting state in one process Desktop apps, low-latency interactive use
Multi-process Backend daemon + window/agent process via IPC protocol Isolation, orchestration, Python-driven sessions

Rendering Safety Principles

  • Fence-aware frame lifecycle
  • Swapchain recreation paths for resize/out-of-date
  • No per-frame pipeline rebuilds in hot paths
  • Validation-focused CI with golden regression tests
For implementation details, see src/render/vulkan and plans/ docs in the repository.
